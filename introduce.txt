项目实施同步（2026-06-23）

当前已完成 Sprint 0 的前后端基础工程：
1. 后端：Spring Boot 3 + JPA + MySQL 8，提供 /api/health、/api/home、注册和登录接口；密码使用 BCrypt 哈希保存，接口具有统一返回体和参数校验。
2. 前端：Vue 3 + Vite，提供响应式首页、后端在线状态、注册和登录界面，并通过 Vite 代理调用后端 API。
3. 数据库：脚本已按 00–08 编号拆分为数据库、用户、小说标签、章节图、评论、竞标、角色和热度模块；具体导入顺序见 db/mysql/README.md。
4. 验收状态：后端已成功连接 MySQL，Maven 打包和 Vue 构建均已通过。若已导入旧版用户表，需先执行 08_align_users_role.sql，将 users.role 调整为 VARCHAR(20)，再启动项目完成注册/登录/健康检查的最终冒烟测试。

Sprint 0 最终验收结果：已完成。
✔ 后端启动且 /api/health 返回 UP
✔ 前端可访问并显示服务在线
✔ 可以成功注册、退出、再次登录
✔ 用户记录持久化到 MySQL 的 users 表

Sprint 0 补充实现（JWT 与前端路由）：
1. JWT：注册和登录成功后签发 HS384 JWT，默认有效期为 120 分钟；令牌含用户名与角色声明。后端以无状态方式校验 Authorization: Bearer <token>，并提供受保护的 /api/auth/me 验证接口。
2. 前端路由：Vue Router 提供 /（首页）、/login（登录）和 /register（注册）路由；前端 API 层会读取并自动携带本地保存的 JWT。
3. 已完成验证：后端 Maven 打包成功；前端 npm run build 成功；H2 临时验证中，登录签发 JWT 后成功访问 /api/auth/me。
